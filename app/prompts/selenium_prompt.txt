Build locators ONLY from attribute values that appear in the

Locator strategy, in order of preference:

1. By.ID           — if the element has an id
2. By.NAME         — if the element has a name
3. By.CSS_SELECTOR — from type/class attributes,
   e.g. submit buttons: "button[type='submit']"
4. By.XPATH with normalize-space() for visible text,
   e.g.: "//button[normalize-space()='Login']"

NEVER use contains(text(), ...) — it breaks on whitespace
and nested elements. Always use normalize-space() instead.

If an element required by a test case is NOT present in the DOM,
use the closest matching element from the DOM INFORMATION instead.

Never write the string REPLACE_WITH_ACTUAL_LOCATOR anywhere.

After clicking a button that triggers navigation or validation,
wait for the result before asserting. Use GENERIC patterns that
work on any website — for example:

start_url = driver.current_url
login_button.click()
WebDriverWait(driver, 15).until(
    lambda d: d.current_url != start_url
    or len(d.find_elements(
        By.CSS_SELECTOR,
        "[class*='alert'], [class*='error'], [role='alert']"
    )) > 0
)

Assertions must verify the test case's expected result
(URL change, error message shown, element visible) —
never assert only that the title is non-empty.

Never hard-code website-specific CSS class names that are not
present in the DOM INFORMATION.
